📖 Production Guide — Get the maximum value from ADTOS

A 10-minute orientation. Read it once; reread sections weekly during your first month.

1. The mental model

ADTOS is a four-layer hybrid orchestration system:

  • L1 Events (Workflows) — when to start work (cron, webhook, command)
  • L2 Orchestration (Agents) — who decides what and audits it
  • L3 Capability (Skills) — the 47 atomic techniques
  • L4 Tools & Data (MCP) — external systems (Notion, Linear, Slack, GitHub)

Treat ADTOS like an operating system, not a checklist. You run apps (agents) that call functions (skills) on data (MCP).

2. The first 60 minutes

  1. Spend 15 min on Insights. The 20 non-obvious patterns explain why most design thinking fails and what ADTOS does differently.
  2. Walk the 7 stages (15 min). Click each stage; note its 4 deliverables and exit metric.
  3. Open the Pentastack matrix (10 min). Notice that each cell is a skill — this is the compounding map.
  4. Run the Synthesizer on a real challenge (15 min). Use a problem you actually have. Step through all 6 stages.
  5. Log the synth run in Decision Journal (5 min). Score quality 1-5; come back in 90 days to score outcome.

6. Anti-patterns to avoid

  • Idea theater. Pretty slides with no evidence. Always tag where each claim sits on the Say-Do-Invest ladder.
  • Skipping Stage 0. Applying Complicated-domain methods to Complex problems is the #1 failure mode.
  • Confusing quality with outcome. A good decision can have a bad outcome. Score them separately.
  • Banned adjectives. "Best-in-class", "world-class", "seamless" — operationalize or strike them.
  • Mismatching velocity to door type. Reversible decisions deserve speed; irreversible ones deserve a written memo and ≥5 reviewers.
  • Vague Deciders. Every brief must name one human Decider. "The team" is not a Decider.
  • Vanity metrics. Active users and signups in the discovery phase are noise. Track validated-uncertainty reduction.
  • Mixing story jobs. One audience, one job per narrative section. Mix them and you lose.

7. The 99th-percentile playbook quality bar

An ADTOS playbook is considered 99th-percentile if and only if:

  • Every framework carries a chapter-level citation
  • Exactly 7 stages with 4 deliverables each
  • ≥5 implementation plans with named Deciders and dated metrics
  • ≥12 reusable templates
  • Pentastack coverage matrix is full (no thin cells)
  • A9 Audit: PASS
  • A10 Evaluation: ≥85/100
  • Zero forbidden adjectives
  • Zero ambiguous pronouns at paragraph start
  • Every metric is quantitative and dated

8. Deploying ADTOS in Claude — the 30-minute setup

  1. Download the plugin: adtos-plugin-v3.1.zip
  2. Create a Claude Project named ADTOS-[program]-2026
  3. Upload all 14 source PDFs to the Project knowledge base
  4. Paste the Master Operating Manifest (Section B from Item 1) into Project Instructions
  5. Install the plugin into Claude Code: claude install adtos-plugin-v3.1.zip
  6. Run /ADTOS:diagnose on your top-of-mind challenge to verify routing
  7. Run /ADTOS:synthesize on a vague topic to verify A11 + A9 + A10 loop
  8. Open the Decision Journal Entry #1 and commit your launch decision

9. Scaling beyond yourself — team adoption

Week 1. Run ADTOS alone on three real challenges. Build confidence in the system.

Week 2. Share the Synthesizer with one trusted peer. Have them run it on a different challenge. Compare results.

Week 3. Pick one weekly meeting (e.g., portfolio review) and replace its agenda with the Pentastack matrix + Q×Q review.

Week 4. Train 3-5 "Innovation Catalysts" (Kelley term) who can run ADTOS in their own programs.

Month 2. Stand up the MCP servers (Decision Registry, Innovation Accounting, Pentastack Audit) so artifacts persist across projects.

Month 3. Run the first quarterly Q×Q Decision Quality Review with the leadership team.

Month 6. ADTOS is the default operating model. New PMs are onboarded with the Production Guide on day one.

10. Ten warning signs you're using ADTOS wrong

  1. You never use the Synthesizer (you're skipping the easiest 10× lever).
  2. The Decision Journal is empty after 30 days (you don't trust yourself, or you're not committing).
  3. You always classify problems as Complex (you're avoiding the Simple/Complicated discipline).
  4. Your Pentastack matrix has gaps in the Evidence row (you're doing idea theater).
  5. Your stories all have the same audience (you're not segmenting).
  6. Your SPADE memos lack pre-mortems (you're skipping the most important step).
  7. You never invoke A9 Audit or A10 Evaluation (you're shipping unaudited work).
  8. You bypass Stage 0 Cynefin (you're applying the wrong method).
  9. You ship without LRI ≥ 80 (you're betting on luck).
  10. You don't run the quarterly Q×Q review (you're not learning).

11. What ADTOS will not do for you

  • It won't replace your judgement — it amplifies it.
  • It won't tell you what to decide — it forces you to articulate why.
  • It won't compensate for missing data — it surfaces what you don't know.
  • It won't make you popular — it makes you precise.
  • It won't speed up bad decisions — it slows down Type 1 (irreversible) ones.

12. Where to go next

You now have the full system. The order of operations:

  1. Today — run the Synthesizer on a real problem (15 min).
  2. This week — install the plugin in Claude and run /ADTOS:full on one program.
  3. This month — log every Type 2 decision in the Journal.
  4. This quarter — run your first Q×Q review.
  5. This year — make ADTOS the default operating model in your org.
