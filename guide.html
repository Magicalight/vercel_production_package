Post-deploy checklist

  1. Deploy the chosen backend target.
  2. Add the real secret as a platform environment variable.
  3. Open the app → Settings → Deployment.
  4. Paste or preset the Backend Base URL.
  5. Click Test backend.
  6. Run a real Synthesizer intake and verify `/health`, `/api/context/build`, and `/api/brief/generate`.

Architecture shift. ADTOS now supports a proper secure-backend pattern. The static HTML app keeps only local workspace preferences, while the live model secret belongs exclusively in the backend environment.

  • Frontend: Settings now includes a Backend Base URL and health-test button.
  • Backend: `GET /health`, `POST /api/context/build`, and `POST /api/brief/generate`.
  • Fallback mode: if no backend or secret is available, ADTOS falls back to local heuristics.
  • Security rule: never paste the real Anthropic secret into the app bundle.

Recommended deployment path

Run the backend on a secure platform with environment variables, set an origin allowlist, then point the app Settings → Backend Base URL to that deployment. Use the built-in health test before enabling real work.

Open Settings from the left sidebar to configure workspace identity, default region, model preferences, API key ID placeholder, endpoint, storage mode, theme, and local-history behavior. This is a UI-only readiness layer; no live key is shipped in the app.

2The four-layer mental model

ADTOS is a four-layer hybrid orchestration system, not a checklist:

LayerRoleAnthropic PrimitiveWhen it runs
L1 EventsTriggers (time, webhook, file, command)Workflows (YAML)Scheduled or event-driven
L2 OrchestrationDecides who does what + auditsAgents (A0..A20)Every workflow tick or direct command
L3 Capability72 atomic skills, one job eachSkills (SKILL.md)Called by agents on demand
L4 Tools & DataI/O with external systemsMCP servers + connectorsCalled by skills when external state needed

Key insight

You run apps (agents) that call functions (skills) on data (MCP). Treat ADTOS like an operating system, not a methodology.

3The Sovereign Context Substrate

🌐 v4.0+ doctrine

Context is not a one-shot input — it is a persistent, queryable, versioned substrate that every agent reads from and writes back to on every call. Without it, every downstream call re-derives context, producing drift.

5The Variable Bank — single source of truth

A17 Variable Mapper publishes the Variable Bank: a versioned, namespaced key-value store that every downstream agent reads as system-prompt prefix.

${VAR.client.name}           "Acme Renewables"            confidence: 5
${VAR.client.industry}       "Renewable Energy / Solar PV" confidence: 5
${VAR.client.region}         "OM"                          confidence: 5
${VAR.industry.cagr_5yr}     0.12                          confidence: 4
${VAR.strategic.priorities}  [6, 7, 8, 10]                 confidence: 5
${VAR.strategic.programs}    ["Tanwee'a","Tashgheel"]      confidence: 5
${VAR.strategic.score}       82                            confidence: 4
${VAR.session.decider}       "COO Ms Al-Habsi"             confidence: 5
${VAR.session.cynefin}       "Complex"                     confidence: 4
${VAR.session.horizon}       "3-12 months"                 confidence: 5
${VAR.corpus.documents}      14                            confidence: 5
${VAR.corpus.tov.formality}  4                             confidence: 4

Every variable has {value, source, confidence_1_5, last_updated_ts, mutable bool}. Bank size ≤ 50KB. Version is bumped on every confirmed change.

6The A20 Context Confirmation Gate

After A17 publishes the bank, A20 generates 5-10 multi-select questions targeting low-confidence fields, source conflicts, and missing-but-critical dimensions. The user answers; the bank is mutated; only then does A0 enter Stage 0.

Standard 8-question set (Region=OM):

  1. Who is the named human Decider?
  2. What is the primary time horizon?
  3. Which North Star Metric will measure success?
  4. Is the core decision reversible within 30 days at <10% cost?
  5. Which Vision 2040 priorities does this program touch? (multi-select)
  6. Which National Program(s) does this anchor to? (multi-select)
  7. What are the stakes of getting this wrong?
  8. What experience altitude are you targeting?

148 anti-patterns + 10 warning signs

The 8 traps

  1. Idea theater. Pretty slides with no evidence. Tag every claim on Say-Do-Invest.
  2. Skipping Stage 0. Applying Complicated methods to Complex problems is the #1 failure mode.
  3. Confusing quality with outcome. Score them separately on Q×Q.
  4. Banned adjectives. "Best-in-class", "seamless", "world-class" — operationalize or strike.
  5. Mismatching velocity to door. Type 1 needs rigor; Type 2 needs speed.
  6. Vague Deciders. Every brief names one human. "The team" is not a Decider.
  7. Vanity metrics. Track validated-uncertainty reduction in discovery, not signups.
  8. Mixing story jobs. One audience, one job per narrative section.

10 warning signs you're using ADTOS wrong

  1. You never use the Synthesizer.
  2. Decision Journal is empty after 30 days.
  3. You always classify problems as Complex (avoiding discipline).
  4. Pentastack matrix has gaps in the Evidence row.
  5. All your stories share one audience.
  6. SPADE memos lack pre-mortems.
  7. You never invoke A9 or A10.
  8. You bypass Stage 0 Cynefin.
  9. You ship without LRI ≥ 80.
  10. You don't run the quarterly Q×Q review.

15The 99th-percentile quality bar

An ADTOS playbook is 99th-percentile if and only if all 12 hold:

  • Variable Bank published with ≥80% coverage and confidence ≥3
  • Vision 2040 alignment score ≥80 (when region=OM)
  • A20 user confirmation received (no skipped gate)
  • Every framework cited with chapter-level source
  • Pentastack matrix fully populated (no empty cells)
  • 5-act session arc time-boxed (Before/During/After)
  • SPADE memo with pre-mortem + 5-bias + EV attached
  • Launch Readiness Index ≥80
  • A9 Audit PASS, A10 Evaluation ≥85
  • Decision Journal entry logged with Q×Q-review-due date
  • Innovation Accounting ledger climb rate >0
  • Visual report package complete: alignment radar + Pentastack heatmap + journey map + KPI dashboard + risk register
